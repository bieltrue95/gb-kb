## Zero Downtime Deploy

```bash
# Criar slot staging
az webapp deployment slot create \
  --resource-group mygroup \
  --name myapp \
  --slot staging

# Deploy pra staging
az webapp deployment source config-zip \
  --resource-group mygroup \
  --name myapp \
  --slot staging \
  --src-path publish.zip

# Testar em staging (URL: myapp-staging.azurewebsites.net)
# ...

# Swap com production (instantâneo, zero downtime)
az webapp deployment slot swap \
  --resource-group mygroup \
  --name myapp \
  --slot staging
```

## GitHub Actions (automático)

```yaml
# .github/workflows/deploy.yml
name: Deploy to Azure

on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup .NET
        uses: actions/setup-dotnet@v3
        with:
          dotnet-version: '8.0'
      
      - name: Publish
        run: dotnet publish -c Release -o publish
      
      - name: Deploy to App Service
        uses: azure/webapps-deploy@v2
        with:
          app-name: myapp
          publish-profile: ${{ secrets.AZURE_PUBLISH_PROFILE }}
          package: ./publish
```

## Azure CLI Manual

```bash
# Build
dotnet publish -c Release -o publish

# Create App Service Plan
az appservice plan create \
  --name myplan \
  --resource-group mygroup \
  --sku S1

  --name myapp \
  --resource-group mygroup \
  --plan myplan

# Deploy

  --resource-group mygroup \
  --name myapp \
  --src-path publish.zip
```

## Setup e Logging

```csharp
var builder = WebApplication.CreateBuilder(args);

// Adicionar Application Insights
var aiOptions = new ApplicationInsightsServiceOptions 
{ 
    InstrumentationKey = builder.Configuration["APPINSIGHTS_INSTRUMENTATIONKEY"],
    EnableAdaptiveSampling = true,
    EnableDependencyTrackingTelemetryModule = true
};

builder.Services.AddApplicationInsightsTelemetry(aiOptions);

// Configurar logging pra Application Insights
builder.Logging.AddApplicationInsights(
    configureTelemetryModule: (module, _) =>
    {
        module.EnableDependencyTrackingTelemetryModule.IsEnabled = true;
    });

var app = builder.Build();
```

## Instrumentação customizada

```csharp
public class OrderService
{
    private readonly TelemetryClient _telemetry;
    private readonly ILogger<OrderService> _logger;

    public OrderService(TelemetryClient telemetry, ILogger<OrderService> logger)
    {
        _telemetry = telemetry;
        _logger = logger;
    }

    public async Task<Order> CreateAsync(CreateOrderRequest req)
    {
        using var operation = _telemetry.StartOperation<RequestTelemetry>("CreateOrder");
        
        try
        {
            _logger.LogInformation("Creating order for {CustomerId}", req.CustomerId);
            
            // Custom properties
            operation.Telemetry.Properties["CustomerId"] = req.CustomerId;
            operation.Telemetry.Properties["Total"] = req.Total.ToString();
            
            var order = new Order { Id = Guid.NewGuid(), Total = req.Total };
            
            // Custom metric
            _telemetry.TrackEvent("OrderCreated", 
                properties: new Dictionary<string, string> { { "CustomerId", req.CustomerId } },
                metrics: new Dictionary<string, double> { { "Total", req.Total } });
            
            return order;
        }
        catch (Exception ex)
        {
            _telemetry.TrackException(ex);
            throw;
        }
    }
}
```

## Connection Strings e App Settings

```bash
  --resource-group mygroup \
  --name myapp \
  --settings \
```

## Auto-scale Rules

```bash
az monitor autoscale create \
  --resource-group mygroup \
  --resource myapp \
  --resource-type "Microsoft.Web/serverfarms" \
  --name scale-rule \
  --min-count 1 \
  --max-count 5 \
  --count 2

# Adicionar regra: scale out se CPU > 70%
az monitor autoscale rule create \
  --autoscale-name scale-rule \
  --resource-group mygroup \
  --condition "Percentage CPU > 70 avg 5m" \
  --scale out 1

# Scale in se CPU < 30%
az monitor autoscale rule create \
  --autoscale-name scale-rule \
  --resource-group mygroup \
  --condition "Percentage CPU < 30 avg 5m" \
  --scale in 1
```

## ⚠️ Armadilhas comuns

❌ **SKU muito pequeno** → Timeouts, CPU throttling

❌ **Sem slots** → Downtime durante deploy

❌ **Sem Application Insights** → Cego em produção

❌ **Connection strings em código** → Security breach

❌ **Sem health checks** → Liveness issues não detectados

❌ **Scaling horizontal sem stateless** → Inconsistência de dados
